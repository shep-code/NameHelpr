<feature>
  <name>Instant Search with Fuzzy Matching</name>
  <files>src/hooks/useSearch.ts, tests/hooks/useSearch.test.ts, src/views/SearchView.tsx, tests/views/SearchView.test.tsx</files>
  <behavior>
**useSearch hook behavior:**
- Input: people array, returns: { query, setQuery, results }
- Empty query → returns all people
- Query "paul" → finds "Paul's Party" context and "Paul Smith" name
- Query "pauls party" (no apostrophe) → finds "Paul's Party" (threshold: 0.4)
- Query "SARA" (wrong case) → finds "Sara Chen" (case insensitive)
- Prioritizes context matches over name matches (weight: 2 vs 1)
- Memoizes Fuse instance (recreate only when people array changes)
- Results update instantly on query change (no debouncing for small datasets)

**SearchView component behavior:**
- Renders search input at top with placeholder "Search by name or context..."
- Shows hint "Start typing to search..." when query is empty and no results
- Displays results as list when query has matches
- Shows "No results for 'query'" when query returns no matches
- Uses inputMode="search" for mobile keyboard with search button
- 16px font size to prevent iOS zoom (per Phase 2 mobile patterns)

**Test cases:**
- useSearch: returns all people when query empty
- useSearch: filters by fuzzy matching on name and context
- useSearch: handles typos (missing apostrophe, wrong case)
- useSearch: prioritizes context over name (search "party" returns contexts first)
- SearchView: renders input and hint when empty
- SearchView: shows results when query matches
- SearchView: shows "no results" when query has no matches
  </behavior>
